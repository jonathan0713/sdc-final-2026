# Radar Multi-Object Tracking Starter Code

This package is a **student starter framework**, not an official scoring baseline and not a complete tracking solution.

The code is intentionally incomplete. It provides:

- data loading utilities
- submission CSV writing utilities
- result format checking
- local evaluation for public development data
- a tracker skeleton with TODO blocks

Students must complete the TODO blocks before obtaining meaningful tracking IDs.

## Important note

The initial `result.csv` generated by the unmodified starter code is only format-valid. It should **not** be treated as a tracking baseline. Meaningful tracking performance requires implementing the detection extraction, tracker update, data association, and cluster-to-track mapping logic.

## Files

```text
main.py                    Main entry point. Reads frames, builds detections, calls tracker, writes result.csv.
sdc_tracker.py             Minimal tracker skeleton. Data association is intentionally incomplete.
sdc_tracking_utils.py      Data loading and CSV writing utilities. Usually does not need modification.
check_result_format.py     Checks whether result.csv follows the required Kaggle submission format.
evaluate_tracking.py       Local MOTA-style evaluator for public development GT only.
TODO_GUIDE.md              Step-by-step guide for completing the TODO blocks.
requirements.txt           Required Python packages.
```

## Task overview

You are given 4D radar point clouds and pre-computed moving cluster masks. Your goal is to assign temporally consistent tracking IDs to radar points across frames.

Radar points contain five values:

```text
x, y, z, range_rate, rcs
```

The input cluster mask uses:

```text
-1   static point
>=0  moving cluster id
```

The output tracking mask should use:

```text
-1   static point
-2   moving point without confirmed track
>=0  confirmed tracking id
```

## Install dependencies

```bash
pip install -r requirements.txt
```

If your files are placed in `starter_code/`, run commands from that folder.

# Competition 1: static ego tracking

## Run on public development sequence

```bash
cd starter_code

python main.py \
  --data-root ../public_dev \
  --seq seq_1 \
  --output-root ../outputs
```

## Check output format

```bash
python check_result_format.py \
  --submission ../outputs/<YOUR_SEQ1_OUTPUT_FOLDER>/result.csv \
  --data-root ../public_dev \
  --seq seq_1
```

## Evaluate on public development GT

```bash
python evaluate_tracking.py \
  --gt ../public_dev/seq_1/gt_answer_seq1.csv \
  --pred ../outputs/<YOUR_SEQ1_OUTPUT_FOLDER>/result.csv \
  --out ../outputs/seq_1_eval.csv
```

## Generate Kaggle submission for seq2

```bash
python main.py \
  --data-root ../test_input \
  --seq seq_2 \
  --output-root ../outputs
```

Submit:

```text
../outputs/<YOUR_SEQ2_OUTPUT_FOLDER>/result.csv
```

## Competition 2 hint

For `seq_3` and `seq_4`, `ego_global_pos.txt` is provided. Students are encouraged to use ego motion information to design ego-aware association costs or motion compensation methods.

## Submission format

A valid submission CSV must contain:

```text
id,mask
0,...
0x,...
0y,...
0z,...
1,...
1x,...
1y,...
1z,...
```

Each frame must have four rows:

1. tracking labels
2. x coordinates
3. y coordinates
4. z coordinates

Use the provided sample submission file as the format reference.
